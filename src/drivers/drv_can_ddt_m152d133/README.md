# 本末 (DDT) M152D133 驱动使用说明

本文档是对 `drv_can_ddt_m152d133` 电机驱动模块的使用说明，适用于通过 SocketCAN 与该型号（或兼容协议的本末电机）进行通信与控制。

## 一、 通信协议基础

*   **物理层协议**：标准 CAN 总线 (SocketCAN)
*   **默认波特率**：1 Mbps
*   **端序**：大端序 (Big-Endian)
*   **通信架构**：异步非阻塞，指令与反馈完全分离。驱动内部通过 epoll/recv 监听反馈报文，状态数据与下发指令在时间上解耦。

## 二、 支持的控制模式

电机固件原生支持四种主动控制模式以及使能/失能状态：

| 枚举值 / 协议值 | 模式名称 | 描述与特性 |
| :--- | :--- | :--- |
| `MOTOR_MODE_IDLE` (0x09) | 失能模式 (Disable) | 电机断电释放，转子可以自由转动。这是驱动初始化的默认状态。 |
| `MOTOR_MODE_OPEN` (0x00) | 电压开环模式 (Open Loop)| 直接下发电压占空比（-32767 ~ 32767），对应最大电压的比例。空载时不推荐使用。 |
| `MOTOR_MODE_TRQ` (0x01) | 电流/扭矩模式 (Current) | 直接控制电机输出扭矩。给定值范围 `-16383 ~ 16383`，对应实际电流 `-55A ~ 55A`。 |
| `MOTOR_MODE_VEL` (0x02) | 速度模式 (Velocity) | 闭环控制电机转速。目标速度下发后，固件内置 PI 控制器维持速度。 |
| `MOTOR_MODE_POS` (0x03) | 位置模式 (Position) | 闭环控制绝对位置。**注意：** 位置环模式空载启动时会发生一次抖动，属正常现象，如有需要可通过 调节 PI 参数改善。 |

## 三、 PI 参数与滤波器设置

除了基础的运动指令，该驱动通过 `motor_set_paras()` 接口透传了底层固件的高级配置寄存器：

### 1. PI 控制参数设置 (标识符: `0x10C`)
电机固件的性格（刚度与响应）可以通过 PI 参数配置。
*   **接口映射**：`BM_M152D133_PARAM_PI` (Enum 值: 7)
*   **数据结构**：包含 `loop_mode` (目标生效环 0~3)、`kp` (比例增益) 和 `ki` (积分增益)。
*   **使用场景**：在空载震荡时，可适当下调 `kp` 解决高频发抖；在重载拖尾时，可提高 `ki` 消除稳态误差。

### 2. 电流滤波/限制系数 (标识符: `0x10D`)
*   **接口映射**：`BM_M152D133_PARAM_CURRENT_FILTER` (Enum 值: 8)
*   **使用场景**：调整电流环低通滤波器的截止频率或限流阈值，以平衡高频 PWM 噪声与系统响应延迟。

### 3. 反馈周期配置 (标识符: `0x10E`)
*   **接口映射**：直接传入地址偏移 `3`。
*   **重要提示**：**必须在初始化后显式将反馈周期设为 `20ms` 或以上**。出厂默认的 `1ms` 上报频率在多电机级联时会瞬间塞满 Linux CAN 缓冲区，导致获取到陈旧的“时空穿梭”数据。

## 四、 提供的核心接口功能

本驱动完美适配了框架层 `motor_ops` 标准结构体，提供以下 C 语言级统一调用接口：

### 1. `motor_alloc_can` & `motor_init_one`
*   **功能**：分配并绑定具体的 SocketCAN 接口（如 `can0`），启动底层接收线程并下发失能指令，清理陈旧的 CAN 队列以准备就绪。

### 2. `motor_set_cmd_one`
*   **功能**：多路复用的指令下发接口。
*   **特性**：
    *   内置状态机管理：如果目标模式与当前模式跨度过大（例如从电流环切位置环），驱动会自动插入 `0x09` 失能并执行 `50ms` 软切等待，防止固件死机断联。
    *   **位置多圈逻辑自解析**：支持超 360° 的无限制连续目标下发，驱动会自动解算 `pos_turns` 并换算为 `0~32767` 的协议单圈目标值。

### 3. `motor_get_state_one`
*   **功能**：非阻塞获取最新一帧电机状态反馈。
*   **数据内容**：
    *   `pos`：绝对物理位置（弧度 Rad），自动拼接多圈数据。
    *   `vel`：实时角速度（Rad/s）。
    *   `trq`：实时输出力矩/电流估算（Nm）。
    *   `err`：错误码掩码（过压、过温、编码器故障等）。

### 4. `motor_set_paras`
*   **功能**：高级透传接口，支持设置 PI 参数、电流滤波、反馈周期等低频操作。

### 5. `motor_free`
*   **功能**：安全退出，下发失能指令并销毁套接字资源，防止电机失控暴走。
