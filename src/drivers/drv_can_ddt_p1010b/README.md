# P1010B_111 电机控制说明

本驱动模块用于支持本末（DDT）P1010B 系列 CAN 接口伺服电机。由于该系列电机通信协议具有特殊的“组帧广播”机制，在使用 SDK 接口时，请务必注意以下设计约束：

## 1. 基础通信与控制参数
- **支持协议**：CAN总线 (CANFD/CAN2.0，标准帧) 及 RS485。
- **额定参数**：额定电压 24VDC，额定母线电流 16A，额定扭矩 20Nm。
- **给定值换算比例**：
  - **电压模式**：给定范围 `-4800 ~ 4800`，对应量程 `-48V ~ 48V`（100 倍率）。
  - **电流（转矩）模式**：给定范围 `-7500 ~ 7500`，对应量程 `-75A ~ 75A`（100 倍率）。
  - **速度模式**：给定范围 `-16000 ~ 16000`，对应量程 `-1600RPM ~ 1600RPM`（10 倍率）。
  - **位置模式**：给定范围 `-5000 ~ 5000`，对应量程 `-50圈 ~ 50圈`（100 倍率，已折算减速比至**输出轴**）。

## 2. 一帧控制多电机（组帧机制）
P1010B 的驱动给定指令采用**一帧报文同时下发多个电机**的格式：
- **`0x32`**：控制 ID 1 ~ 4 号电机。
- **`0x33`**：控制 ID 5 ~ 8 号电机。

在 SDK 底层驱动实现中，为了防止用户层并发调用 `set_cmd` 导致 CAN 总线冲突与同组报文覆盖，驱动内引入了 `g_bus_states` 缓存机制，对发往同一分组的电机指令进行拦截与合并组帧。

## 3. `set_cmd` 的单参数约束
受限于上述的控制帧结构设计（每 16-bit 槽位仅能容纳一个电机的核心给定值），**本类型电机的 `set_cmd` 接口不支持同时传递多个参数来控制单个电机**。
- 例如：在位置模式下，`set_cmd` 只能下发“目标位置（`pos_des`）”，**无法**在同一帧中附带下发前馈速度、前馈力矩或加速度限制等辅助参数。
- SDK 会根据当前所处的工作模式，自动选择 `cmd` 结构体中的某一核心参数（`pos_des`、`vel_des` 或 `trq_des`）按照上述的“换算比例”编码进控制帧发出。

## 4. 辅助参数的配置与读取
对于诸如加减速度限制、PID 增益调节、工作模式切换等辅助需求，以及底层状态的按需查询，需依赖专用接口：
- **写入参数（`set_paras`）**：
  - 底层基于 `0x36` 通信原语。支持精确地向指定 ID 的特定寄存器写入 32-bit 数据。
  - **示例**：如需在位置模式下限制电机的最大运行速度，不可依赖 `set_cmd`。必须先通过 `set_paras` 写入速度限制配置，再通过 `set_cmd` 下发位置目标。
- **读取参数（`get_paras`）**：
  - 底层基于 `0x35` 查询原语。支持动态获取底层核心参数的值。
  - **常见用法**：查询 `0x09` 以获取当前实际工作模式（0=电压，2=电流，3=速度，4=位置）；查询 `0x0D` 以获取当前单圈绝对位置；查询 `0x0B` 获取多圈累计数值以实现绝对零点同步。

## 5. 电流环（力矩模式）制动与停机注意事项
在电流模式下，如果给电机持续下发较大电流（如 > 1A）进行运动测试，当需要电机停止（如到达指定位置）时，需特别注意控制策略

- **启动静摩擦与“磁驻点”现象**：在进行模式切换（特别是从位置环等强闭环模式切回再切入电流环前，经历了 IDLE 瞬间失能）时，电机转子往往会瞬间落入最近的磁驻点（即阻力最大的驻留位置）。在这种状态下，下发 1A 的目标电流有时会因为无法冲破起步静摩擦力而导致电机无法被驱动。若遇到此现象，请以 0.1A 为梯度适当增加给定电流即可克服摩擦，切勿一次性盲目给定过大电流。

- **惯性滑行（过转）现象**：在电机控制中，`IDLE` 状态意味着彻底断电失能，驱动器不再施加任何制动阻尼。如果直接失能，电机会因为物理惯性继续自由滑行（Freewheel）。额定下发的电流越大、积攒的速度越高，电机滑行的距离和时间也就越长。

- **推荐的制动与停止逻辑**：
  - **位置控制/低速场景**：如果在电流环下有目标位置精度要求，应在逻辑中显式地**将速度（V）钳位到 0**（切换为 `MOTOR_MODE_VEL` 且 `vel_des = 0.0f`）或切入位置环。待主动电磁力矩将电机完全刹停后，再安全地下发 `IDLE`。
  - **大电流/高速场景的安全制动**：当运行电流（动能）过大时，**强烈不建议**直接将速度突变置零（V=0）。这种刚性刹车可能产生巨大的瞬态反激电流，导致驱动器过流保护甚至损坏硬件。正确的做法是采取**力矩/电流平滑递减（Ramp down）**或**速度减速规划曲线**柔性刹车，待速度显著降低后再切入 `IDLE`。

  经试验，在 1A 电流下，到达目标位置进行速度钳位，已有较强的制动效果，超过 1A 可能有安全隐患，请谨慎使用。

## 6. 测试用例编译控制
为了避免构建产物过度冗余，工程的 `CMakeLists.txt` 对 P1010B 的测试代码做了按需编译的开关控制：
- **默认构建**：常规编译时，仅保留综合多模式切换的验证主程序 `test_motor_can_ddt_p1010b`。
- **扩展测试**：若在开发调试过程中需要编译并运行 `test_p1010b`、`test_p1010b_state` 以及 `test_p1010b_pos_traj` 等独立的子功能单元测试，请在发起 CMake 配置时手动开启编译标志：
  ```bash
  # 开启附加测试编译
  cmake -DBUILD_P1010B_EXTRA_TESTS=ON ..
  ```
