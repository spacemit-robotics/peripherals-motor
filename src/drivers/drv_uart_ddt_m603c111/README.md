# M0603C_111 电机控制说明

本驱动模块用于支持本末（DDT）M0603C 系列 UART 接口伺服电机。在使用 SDK 接口时，请务必注意以下设计约束：

## 1. 基础通信与控制参数
- **支持协议**：UART 串口通信 (115200 波特率，8数据位，1停止位，无校验，一问一答反馈机制，最高速率 500Hz)。
- **给定值换算比例**：
  - **开环（电流环）模式**：给定范围 `-32767 ~ 32767`，对应量程 `-8A ~ 8A`。
  - **速度模式**：给定范围 `-2100 ~ 2100`，对应 `-210rpm ~ 210rpm`（有符号 16 位）。
  - **位置模式**：给定范围 `0 ~ 32767`，对应 `0° ~ 360°`（无符号 16 位单圈绝对位置）。

## 2. 状态控制与模式切换
- **上电即使能**：系统上电即可通过 `0xA0` 原语指令切换模式 (`0x00` 开环, `0x02` 速度环, `0x03` 位置环) 进行使能驱动。
  - **注意**：切换到位置环模式前，要求电机的当前转速必须低于 10rpm，否则可能引发切换异常。
- **失能释放**：当 SDK 下发 IDLE 时，驱动底层会主动将电机**切换为开环（电流）模式，并下发 0A 电流**。此时电机将完全失去电磁力矩，处于**无阻尼的自由滑行（释放）状态**，可以通过外力轻松转动。
  - **重要提醒**：此失能方式并不是物理上的断电失能，驱动器内部控制环路仍在正常运行，该设计仅仅是为了使电机快速进入顺从状态，因此**不适合作为长时间待机状态使用**。

## 3. `set_cmd` 参数约束
受限于底层的通信帧格式（`0x64` 控制帧）：
- 驱动会根据当前的工作模式，自动选择 `cmd` 结构体中的某一核心参数（`pos_des`，`vel_des` 或 `trq_des`），并按换算比例编码为 16-bit 给定值下发给电机。
- **辅助运动参数**：底层帧预留了加速时间（速度模式下生效，每 1rpm 加速时间/ms）和制动刹车标志位，但当前 SDK 层面的 `set_cmd` 主要负责主给定的传递。

## 4. 辅助参数与状态读取 (`get_paras` / `set_paras`)
诸如获取实际绝对编码器数值、里程信息、软硬件版本等，需通过下发专用查询帧，并通过 SDK 的异步读取接口获取。
- **里程与位置查询 (`0x74` 帧)**：
  - 通过 `motor_get_paras(dev, (const void*)0x74, ...)` 可获取：
    - 传入 `uint16_t` 大小：获取当前原始绝对位置值 (`pos_raw`，`0~32767` 对应 `0~360°`)。
    - 传入 `float` 大小：获取多圈里程圈数（底层为 32 位有符号计数，范围 ±2,147,483,647，**重新上电会清零**）。
- **工作模式查询 (`0x75` 帧)**：返回内部当前的实际运行模式。
- **版本号查询 (`0xFD` 帧)**：可读取出厂年月日、电机型号以及软硬件版本。
- **电机 ID 查询与设置**：
  - 查询 ID：使用特定的 `0xC8` 查询指令。
  - 设置 ID：需确保总线上只有一个电机，发送 `0xAA 0x55 0x53` 等特征字节的改写帧。注意：每次上电只允许设置一次，接收到 5 次 ID 设置指令后方可生效。

## 5. 硬件阈值与安全保护
底层硬件自带以下物理保护机制，当触发时电机会切断输出进入保护状态：
- **母线过流保护**：2.8A（发生过流后触发停机保护，5秒后自动解除）。
- **电机过温保护**：80℃（温度低于阈值5℃，即75℃以下后自动解除保护）。
- **堵转保护**：持续堵转时间超过 5 秒触发保护，5秒后自动解除。

## 6. 已知问题与位置闭环漂移警告（重要）

> [!WARNING]
> **位置模式下的积分漂移异常与归位反转风险**
> 
> 在上电连续运行、历经多模式切换且累积旋转上百圈后，若尝试切回位置环模式，可能会出现严重的安全异常：
> **模式切换指令下发后、控制指令下发前，电机发生反方向的剧烈旋转（朝着初始目标位置反转“归位”）**。
> 
> - **根本原因**：经多次验证排查，这是由于电机底层固件的位置闭环积分器在长时间、多圈高速旋转（特别是在速度环或开环模式下）过程中，累积了逻辑误差并产生了积分漂移。导致内部逻辑坐标系与物理绝对编码器的值严重脱节。
> - **使用建议**：如果您的应用场景强依赖于**位置环**控制且需要较高的绝对精度，**强烈建议在经历过长时间其他模式（如开环或速度环）的连续运行后，切断电机供电并重启**。断电重启能够重置底层寄存器的积分状态，避免直接切换带来的反转失控隐患。

## 7. 测试用例编译控制
为了验证电机各模式特性及追踪漂移问题，驱动目录下包含多个功能单元测试（如 `test_m603c_pos`、`test_m603c_open` 等）。
如需编译并运行这些子功能测试用例，请在发起 CMake 配置时手动开启全量驱动测试标志：
```bash
cmake -DBUILD_DRIVER_EXTRA_TESTS=ON ..
```
