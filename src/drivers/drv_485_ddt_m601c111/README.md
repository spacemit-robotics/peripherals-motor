# M0601C111 电机控制说明

本驱动模块用于支持本末（DDT）M0601C111 系列 RS485 接口伺服电机。驱动通过统一的 motor SDK 接口提供电流环、速度环和位置环控制，并按照设备协议完成 10 字节请求/响应通信。

## 1. 基础通信与控制参数

- **通信方式**：RS485，总线一问一答。
- **波特率**：115200。
- **数据格式**：8 数据位、无校验、1 停止位（8N1）。
- **报文长度**：请求和响应均为 10 字节。
- **最高通信速率**：500Hz。
- **电机 ID**：由 SDK 创建电机时传入，驱动将其作为每帧 `DATA[0]`。
- **默认工作模式**：速度环模式；后续由 `set_cmd` 根据 SDK 工作模式自动切换控制环。


## 3. 控制帧格式

### 3.1 模式切换与使能/失能

模式切换使用命令 `0xA0`：

| 字节 | 含义 |
| --- | --- |
| `DATA[0]` | 电机 ID |
| `DATA[1]` | `0xA0` |
| `DATA[2]~DATA[8]` | 置 0 |
| `DATA[9]` | 模式值：`0x01` 电流环，`0x02` 速度环，`0x03` 位置环 |

按照现有设备说明，`DATA[9]` 为模式值，不计算 CRC。驱动发送该帧后等待约 2ms，再更新本地模式缓存。

### 3.2 驱动电机

驱动控制使用命令 `0x64`：

| 字节 | 含义 |
| --- | --- |
| `DATA[0]` | 电机 ID |
| `DATA[1]` | `0x64` |
| `DATA[2]~DATA[3]` | 电流/速度/位置给定，高字节在前 |
| `DATA[4]~DATA[5]` | 协议预留给定字段，当前驱动置 0 |
| `DATA[6]` | 加速时间，当前驱动置 0（最快加速） |
| `DATA[7]` | 刹车控制，当前驱动置 0（不刹车） |
| `DATA[8]` | 置 0 |
| `DATA[9]` | 对 `DATA[0]~DATA[8]` 计算 CRC8 MAXIM |

CRC8 MAXIM 的初值为 `0x00`，多项式实现为右移形式 `0x8C`。

## 4. 反馈帧格式与状态换算

收到 `0x64` 的响应后，驱动校验 CRC，并确认响应模式与当前目标模式一致；校验失败或通信超时不会覆盖最近一次缓存状态。

| 字节 | 含义 |
| --- | --- |
| `DATA[0]` | 电机 ID |
| `DATA[1]` | 当前模式值 |
| `DATA[2]~DATA[3]` | 实际转矩电流，有符号 16 位，高字节在前 |
| `DATA[4]~DATA[5]` | 实际速度，有符号 16 位，高字节在前，单位 rpm |
| `DATA[6]~DATA[7]` | 实际位置，无符号 16 位，高字节在前 |
| `DATA[8]` | 故障码 |
| `DATA[9]` | CRC8 MAXIM |

驱动向 `motor_state` 提供以下值：

- **转矩 `trq`**：`current_raw / 32767 × 8`，单位 A。
- **速度 `vel`**：`speed_raw × 2π / 60`，单位 rad/s。
- **位置 `pos`**：`position_raw / 32767 × 2π`，单位 rad，范围约 `0~2π`。
- **错误 `err`**：直接返回 `DATA[8]` 故障码。

## 5. 其他反馈查询

其他反馈使用命令 `0x74`：

### 5.1 查询请求

| 字节 | 含义 |
| --- | --- |
| `DATA[0]` | 电机 ID |
| `DATA[1]` | `0x74` |
| `DATA[2]~DATA[8]` | 置 0 |
| `DATA[9]` | CRC8 MAXIM |

### 5.2 查询响应

| 字节 | 含义 |
| --- | --- |
| `DATA[0]` | 电机 ID |
| `DATA[1]` | 当前模式值 |
| `DATA[2]~DATA[3]` | 实际转矩电流 |
| `DATA[4]~DATA[5]` | 实际速度 |
| `DATA[6]` | 绕组温度 |
| `DATA[7]` | U8 单圈位置，0~255 对应 0~360° |
| `DATA[8]` | 故障码 |
| `DATA[9]` | CRC8 MAXIM |

当前驱动已实现 `get_paras` 对地址 `0x74` 的支持：

- 当 `data_len == sizeof(uint16_t)` 时，直接返回最近一次 `0x64` 反馈中的原始位置值。
- 当 `data_len == 1` 时，发送 `0x74` 查询帧并解析查询响应中的 U8 位置值。
- 其他地址或数据长度返回失败。

## 6. 参数与特殊指令

### 6.1 加速时间

在 `0x64` 控制帧的 `DATA[6]` 设置加速时间：

- 写 `0` 表示最快加速，即设备默认加速时间。
- 数值越大，加速越慢。
- 当前驱动固定发送 `0`，如需调节该参数，需要扩展驱动接口或控制帧组装逻辑。

### 6.2 刹车控制

在速度环模式下，`0x64` 控制帧的 `DATA[7]` 支持刹车：

- `0xFF`：执行刹车。
- `0x00`：不刹车。

当前驱动固定发送 `0x00`，SDK 的 `set_cmd` 尚未提供独立刹车参数。

### 6.3 电机 ID 设置

设置 ID 前应保证 RS485 总线上只有一个电机。设备要求连续接收 5 次设置命令后才生效：

```text
AA 55 53 [ID] 00 00 00 00 00 00
```

其中 `DATA[3]` 为目标 ID。该指令不属于当前驱动的自动控制流程，应在确认总线安全后单独发送。

### 6.4 电机 ID 查询

ID 查询指令如下：

```text
C8 64 00 00 00 00 00 00 00 DE
```

该指令将 `DATA[0]` 固定为 `0xC8`，末字节为协议给出的校验值。

## 7. 通信与停机注意事项

- 驱动使用一问一答方式，每次控制发送 10 字节请求，并等待完整的 10 字节响应；响应不完整或超时会返回通信失败。
- 串口初始化为阻塞读模式，设备路径无法打开、串口属性配置失败时，电机创建或初始化失败。
- `MOTOR_MODE_IDLE` 在当前实现中会切换到电流环并发送 0A，不等同于直接断开串口或释放设备。
- 运行中的电机不建议直接从较大电流给定切换到失能状态。应先将电流平滑降低，或切换到速度环并将速度给定降为 0，待电机停止后再进入停机流程。
- 电机具有母线过流、相电流、过温和堵转保护：母线过流阈值约 3A，相电流保护阈值约 4.6A，过温阈值约 80℃，堵转持续约 5s 可能触发保护。具体保护行为以设备实际配置和说明书为准。

## 8. 测试用例编译控制

在 `components/peripherals/motor/CMakeLists.txt` 中，M0601C111 驱动由 `drv_485_ddt_m601c111` 选项控制：

- 未指定驱动列表时，自动检测并编译该驱动（目录存在时）。
- 指定驱动列表时，使用 `drv_485_ddt_m601c111` 
- 驱动存在且启用时，编译基础验证程序 `test_motor_uart_ddt_m601c111`。
- 开启附加测试时，还会编译 `test_m601c` 和 `test_m601c_pos`。

示例配置参数：

```bash
cmake -DSROBOTIS_PERIPHERALS_MOTOR_ENABLED_DRIVERS=drv_485_ddt_m601c111 \
	-DBUILD_DRIVER_EXTRA_TESTS=ON ..
```

测试程序需要连接实际 RS485 设备，并提供正确的串口设备路径和电机 ID；未连接硬件时只能验证编译和链接，不能验证通信时序、CRC 及电机响应。
